Sketch A — single-stream-as-event-log. An SSE exchange occupies one row in the request list, with a live ● live · sse badge, an event count, and a "time since last event" stamp. The row stays pinned to its arrival position; new HTTP exchanges still land above it in time order, but the row itself updates in place rather than thrashing the list. The response pane is replaced by a two-tier stream viewer: a virtualized event log on top, and a per-event detail panel below. Each log row shows index, time-since-previous, the SSE event: type, and a one-line preview of the data (the leading delta string for text deltas; key fragments for structural events). The selected row's full payload renders as pretty JSON underneath. Affordances built around the live nature: a follow toggle keeps the log auto-scrolled to the tail; a sticky "↓ following latest" pill lets you re-engage when you've scrolled away. The trail row at the bottom signals "still streaming" without requiring a scroll. A filter strip (all / deltas / non-delta / errors) lets you collapse the noise from ping/content_block_delta when you only want to inspect structural events. Tradeoffs: the response pane now has two rendering modes (single-blob JSON vs. event log), driven by Content-Type. It scales well to thousands of events via virtualization, but it puts the assembled message — the thing a human actually wants to read — out of view. That's what Sketch E addresses.
