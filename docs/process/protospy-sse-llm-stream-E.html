Sketch E — reconstructed view, with raw events available behind a tab. When protospy recognises a known streaming protocol (Anthropic /v1/messages, OpenAI /v1/chat/completions, Vertex, etc.) by Content-Type plus URL signature, it assembles the deltas into the artefact a human actually wants to read: the message text, the role, the stop reason, the usage block. Tabs above let you flip between reconstructed, raw events (Sketch A's event log), and wire format (the byte-exact data:/event:/id: lines). What the reconstructed pane gives you: a summary header (model · role · stop reason · message id), a progress bar that fills as the stream completes, and one section per content_block. Text blocks render as wrap-flowed prose with markdown-ish emphasis; tool-use blocks would render as JSON forms; thinking blocks would be collapsible. A live cursor sits at the tail while the stream is open, and the stop-reason chip flips from gold ("streaming") to green ("end_turn") on close. Usage and timing are pulled from message_delta events as they arrive. "Show delta seams" is an optional toggle: it underlines the text in faint gold at the boundaries between content_block_delta events, so you can audit exactly which tokens came from which event without leaving this view. For deeper inspection you switch to "raw events" — the same UI as Sketch A. Tradeoffs: protocol-aware. The reconstructor needs to know about each provider's event vocabulary; unknown protocols fall back to "raw events" automatically. The win is large for the dominant use case (LLM streams) — you read the answer the way the API consumer's UI would render it, while keeping byte-level access one click away. This pairs well with Sketch A: A is the universal default; E layers on top when the protocol is recognised.
