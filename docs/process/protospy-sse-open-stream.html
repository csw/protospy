protospyv0.1.0-dev
Open-ended SSE — events interleaved with HTTP exchanges, chronologically. Each SSE event becomes a first-class row in the main list, slotted into time order with regular HTTP exchanges. Two open streams (/api/chat/stream and /api/metrics/live) are tracked simultaneously and given distinct accent colors (gold and teal here); each event row carries a colored left bar that visually links it to its parent stream. The open streams legend at the top of the list names the streams and shows their event count and time-since-last so you can see them at a glance even when their bottom-most rows have scrolled out of view. Two row shapes: exchange rows look exactly as before (method · status · path · timing). Event rows are denser — evt · <event-name> · data preview · time — and indented past the stream's accent stripe. The originating stream-open request appears as its own row at its arrival timestamp (the bottom two rows), so you can scroll to "when this stream started" naturally. Selecting an event opens a stream-aware detail view: a context bar that names the parent and offers a quick "jump to stream ↑" link; a structured fields panel for the SSE wire-level metadata (event, id, retry, plus derived timing — since-previous, since-stream-open); a sibling strip for jumping forward and back through events within the same stream without losing your place in the chronological list; and the same wire/JSON body view used for ordinary responses. Filtering and noise control: the toolbar's filter input would accept tokens like stream:a, event:message, or !event:typing to suppress chatty stream noise without hiding the underlying exchanges. A high-volume metrics stream can be collapsed to a "62 metric events …" group row when it dominates the view. Why this works for open-ended streams (and not for LLM streams): here, each event is semantically meaningful on its own — a chat message, a presence change, a metric tick. Browsing them is the point. For an LLM completion, individual content_block_delta events aren't independently meaningful — a list of 200 fragments is noise, not signal — which is why Sketches A and E keep those events scoped inside one exchange row instead.
